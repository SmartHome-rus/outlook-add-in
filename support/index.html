Test Word Checker - Support

Welcome to the support page for the Test Word Checker Outlook add-in. This add-in helps prevent accidental sending of test emails by scanning for the word "test" and prompting for confirmation.

Frequently Asked Questions

Q: How does the add-in work?
A: When you click "Send" in Outlook, the add-in automatically scans your email body for the word "test" (case-insensitive). If found, it shows a confirmation dialog with options to "Send Anyway" or "Discard & Edit".
Q: What happens if I'm offline?
A: The add-in is designed to never block your emails when you're offline. If no internet connection is detected, your email will be sent normally to your Outbox.
Q: Which versions of Outlook are supported?
A: The add-in works with:
  • Outlook Classic on Windows (with WebView2)
  • New Outlook for Windows and Mac
  • Outlook on the web (OWA)
Q: The add-in isn't working. What should I check?
A: Please verify:
  • The add-in is properly installed and enabled in Outlook settings
  • You have the required permissions (ReadWriteItem)
  • Your Outlook version supports the Mailbox API 1.10 or later
  • Check browser console for any JavaScript errors
Q: Can I customize which word triggers the warning?
A: Currently, the add-in is configured to scan for the word "test". To change this, you would need to modify the source code in the commands.js file and update the regular expression.
Q: Is my email content private?
A: Yes, the add-in only reads your email content locally in your browser/Outlook client to scan for the target word. No email content is transmitted to external servers.
Important: This add-in is designed as a safety net to prevent accidental test emails. It should not be relied upon as the sole method for email validation in critical business processes.

Troubleshooting

Add-in Not Loading

  1. Check that all resource URLs in the manifest are accessible
  2. Verify GitHub Pages is serving files over HTTPS
  3. Clear your browser cache and restart Outlook
  4. Check if popup blockers are interfering with the dialog

Event Not Triggering

  1. Ensure the manifest has the correct OnMessageSend event configuration
  2. Verify the function name matches between manifest and JavaScript code
  3. Check that Office.js is loading properly in the function file

Dialog Issues

  1. Confirm the dialog HTML file is accessible via HTTPS
  2. Check browser console for JavaScript errors in the dialog
  3. Verify Office.js is properly initialized in the dialog

Technical Information

Required Office.js Version: 1.10 or later
